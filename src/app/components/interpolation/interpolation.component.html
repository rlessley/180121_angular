String Interpolation

Angular uses a concept called data binding. This boils down to how we can connect out front end (View) to our backends. (Components) Any data in our components needs to have a way to be transferred to the view. One way to do this, is to use String Interpolation

The value of some componentVariable is: {{componentVariable}}

The above example demonstrated pulling information from from the component directly. We took the data and displayed it on the html. Using the double brackets as provided by angular we can display it to the view. The transfer of data from the component to the HTML is called model.

We can use interpolation like a form of expression language. You can evaluate entities within it.
2 + 2 = {{2 + 2}}...

The author of this site is: {{componentObject.author}}

This was an example of one-way data binding. It is only capable of bringing data from the component to the view. Not vice versa.

Another data-binding option we have is called 'property binding'

Property binding is technique where we can make the value of some element property equal to a result as defined in the typescript file. String interpolation is useful for evaluating data from the backend and displaying it on the front end. Property binding is used to take values from the backend and apply them as properties for the elements.

EVENT BINDING
Event binding is a form of one-way data binding that goes from the frontend to the components. We take action from the html and have it call a function in the component. In order to bind to an event, we use parenthesis'.
